# 基金模拟投资助手 — 需求分析

> 文档版本：v1.0
> 阶段：需求分析
> 技术栈角色：前端主导（React + Vite + TypeScript on Cloudflare Pages） + Cloudflare Workers（数据代理/计算）

---

## 1. 背景与目标

### 1.1 背景
个人投资者在真实买入基金前，往往需要一个"沙盒"环境来：
- 模拟买入/卖出/转换等操作，观察组合表现，而不投入真实资金；
- 验证自己的买卖策略在历史行情下是否有效；
- 实时跟踪持仓基金的盘中估值，辅助决策。

市面上的基金 App 多以"真实交易 + 资讯"为核心，缺少**自由模拟 + 策略回测 + 多数据源对比**的轻量工具，且数据多与账号强绑定。本项目面向"无账号、数据本地化、可自由导入导出"的个人工具场景。

### 1.2 项目目标
构建一个可部署在 Cloudflare Workers & Pages 上的基金模拟投资工具，支持：
1. 模拟持仓操作（买入、卖出、转换等）；
2. 多个模拟持仓集合，支持 Base64 编码字符串导入/导出；
3. 持仓基金实时估值查看，支持多数据源切换（含自建"公开持仓估算"数据源）；
4. 自定义买卖策略 + 策略集合 + 基于历史数据的选定区间收益回测；
5. 策略集合的 Base64 导入/导出；
6. 个人数据保存在浏览器 localStorage。

### 1.3 演进目标（影响架构，非初期交付）
后续需支持**微信小程序**接入。这要求：
- 核心业务逻辑（持仓计算、估值、回测）与 UI/平台解耦，做成平台无关的 TS 核心库；
- 数据存储做抽象层（Web 用 localStorage，小程序用 `wx.setStorageSync`）；
- 数据获取统一走 Workers API，Web 与小程序复用同一套接口。

## 2. 用户角色与使用场景

### 2.1 用户角色
| 角色 | 描述 | 初期是否支持 |
|------|------|------------|
| 个人投资者 | 使用工具进行模拟练习、策略验证，数据存本地 | 是 |
| 小程序用户 | 同一用户在微信小程序内使用，数据存小程序本地存储 | 演进期 |

初期**无账号体系、无服务端用户数据**，所有个人数据本地化。

### 2.2 核心使用场景
- **场景 A：模拟练手**：新手创建一个"练手组合"，用 10 万模拟资金买入若干基金，过几天看盈亏。
- **场景 B：方案对比**：创建多个持仓集合（如"稳健组合""激进组合"），分别管理、对比表现。
- **场景 C：数据迁移/分享**：把某个组合导出为 Base64 字符串，换台电脑或分享给朋友导入。
- **场景 D：盘中跟踪**：交易时段查看持仓基金实时估值，切换不同数据源对比估算差异。
- **场景 E：策略回测**：定义"定投 + 止盈"策略，组合成策略集，选 2020-2023 区间，回测收益。

## 3. 功能需求

### 3.1 模拟持仓操作（需求点 1）

#### 3.1.1 持仓集合与账户
- 一个"持仓集合"代表一个独立的模拟账户，包含：初始资金、可用现金、持仓明细、操作流水。
- 支持设置初始模拟资金。

#### 3.1.2 基金操作类型
| 操作 | 说明 | 关键参数 |
|------|------|---------|
| 买入 | 用现金按金额申购基金 | 基金代码、买入金额、成交日/净值、申购费率 |
| 卖出 | 按份额或金额赎回 | 基金代码、卖出份额/比例、净值、赎回费率 |
| 转换 | 从基金 A 转换到基金 B | 源/目标基金、转换份额、转换费率 |
| 分红（可选） | 现金分红 / 红利再投 | 基金代码、分红方案 |

- 需正确处理基金交易规则：
  - **份额 = 金额 / 申购日净值**（考虑申购费）；
  - 基金按"未知价"成交，需明确以哪一日净值结算（默认下一交易日净值，可简化为最新净值，作为待确认项）；
  - 费率：申购费、赎回费（按持有时间分档）、转换费，需可配置，提供默认值。

#### 3.1.3 持仓计算
- 实时计算：持仓市值、持仓成本、持仓收益、收益率、当日盈亏（基于估值）。
- 组合层面：总资产 = 可用现金 + 持仓市值；总收益、总收益率。

#### 3.1.4 操作流水
- 每笔操作记录留痕（时间、类型、金额、份额、净值、费用），可查看历史。

### 3.2 多持仓集合 + 导入导出（需求点 2）
- 支持创建、命名、删除、切换多个持仓集合。
- **导出**：将单个持仓集合（含基础信息、持仓、流水）序列化为 JSON → Base64 字符串，供用户复制。
- **导入**：粘贴 Base64 字符串 → 解码 → 校验 → 生成新持仓集合。
- 导入需做：格式校验、版本兼容校验、冲突处理（重名时如何处理——待确认，建议生成副本）。
- 数据结构需带 `schemaVersion`，便于未来兼容。

### 3.3 实时估值 + 多数据源（需求点 3）

#### 3.3.1 数据源抽象
定义统一的数据源接口，支持运行时切换。至少包含：
| 数据源 | 类型 | 说明 |
|--------|------|------|
| 接口数据源 A | 第三方接口 | 通过 Workers 代理抓取公开估值接口（如天天基金估值） |
| 接口数据源 B | 第三方接口 | 备用第三方估值源，用于对比/容灾 |
| 自建估算源 | 自算 | 基于基金公开持仓（季报/重仓股）+ 实时股票行情，自行加权计算估值 |

- 用户可在每只基金或全局层面选择使用哪个数据源。
- 支持多个数据源结果并列对比展示。

#### 3.3.2 接口数据源（经 Workers 代理）
- 由于浏览器跨域 + 第三方接口非官方，需用 Cloudflare Workers 做代理转发、缓存、限流、字段标准化。
- Workers 输出标准化的估值 DTO（基金代码、估算净值、估算涨跌幅、估算时间）。

#### 3.3.3 自建"公开持仓估算"数据源
- 抓取基金公开持仓数据（定期报告披露的重仓股/全部持仓）。
- 抓取持仓个股的实时行情。
- 按持仓权重加权计算基金的实时估算涨跌幅与估算净值。
- 需说明局限：持仓为季度披露、有滞后；重仓股仅覆盖部分仓位（需做未覆盖部分的处理策略，待确认）。

#### 3.3.4 估值展示
- 估值刷新（手动 + 可选自动轮询，注意 Workers 调用频率与缓存）。
- 标注估值时间、数据源、是否为交易时段。

### 3.4 自定义策略 + 策略集 + 回测（需求点 4）

#### 3.4.1 策略定义
- 用户可定义买入/卖出策略，策略由"触发条件 + 执行动作"组成。常见类型：
  - 定投（按周期定额买入）；
  - 阈值买入（如跌幅达到 X% 买入）；
  - 止盈/止损卖出（收益率达到 +X% / -Y% 卖出）；
  - 网格（区间内分档买卖）——视复杂度可作为进阶。
- 策略需可配置参数（标的基金、周期、阈值、金额/比例等）。
- 具体支持哪些策略类型、是否需要"自定义表达式/脚本"为待确认项；建议初期提供**预置策略模板 + 参数化**，不做自由脚本。

#### 3.4.2 策略集
- 多个策略可组合成一个"策略集"，作用于一个标的组合。
- 需定义策略冲突/优先级规则（如同日既触发买入又触发卖出）——待确认。

#### 3.4.3 回测
- 输入：策略集、标的基金、回测区间（起止日期）、初始资金。
- 数据：使用接口查询的历史净值数据（经 Workers 拉取并缓存）。
- 输出：区间收益、收益率、年化、最大回撤、操作流水、净值曲线（与基准对比，如标的基金本身的买入持有收益）。
- 回测引擎需基于历史净值逐日（或按交易日）模拟策略触发与交易。

### 3.5 策略集导入导出（需求点 5）
- 与 3.2 一致：策略集序列化 → Base64 导出 / 导入，带 `schemaVersion` 与校验。

### 3.6 数据持久化（需求点 6）
- 个人数据（持仓集合、策略集、用户偏好/数据源设置）保存在浏览器 localStorage。
- 存储层做抽象（StorageAdapter 接口），Web 实现用 localStorage，为小程序预留实现位。
- 需考虑 localStorage 容量上限（约 5MB）：历史回测数据等大对象不长期落地，或做精简存储。

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 部署 | 前端部署 Cloudflare Pages；数据代理/计算部署 Cloudflare Workers |
| 兼容性 | 现代浏览器（Chrome/Edge/Safari 近版本）；为微信小程序预留架构 |
| 性能 | 估值刷新与回测计算在前端/Workers 可接受时延内完成；接口加缓存降低第三方压力 |
| 可移植 | 核心业务逻辑与平台无关，UI 层与逻辑层解耦 |
| 数据安全 | 无服务端存储个人数据；导入数据需校验，避免脏数据/注入 |
| 健壮性 | 第三方数据源不可用时降级/切换；估值/回测结果标注数据来源与时间 |
| 合规 | 明确标注"模拟数据、估值非官方、仅供学习参考，不构成投资建议" |
| 成本 | 控制在 Cloudflare 免费/低成本额度内（Workers 请求数、缓存策略） |

## 5. 约束与边界

### 5.1 范围内（初期）
- Web 端（React SPA）完整实现 6 项功能。
- Workers 提供：估值代理、历史数据代理、基金公开持仓抓取、自建估值计算。

### 5.2 范围外（初期，后续演进）
- 微信小程序客户端实现（仅做架构预留）。
- 真实交易、真实资金、账号体系、多端云同步。
- 投资建议、智能推荐等。

### 5.3 关键风险
- **第三方数据源稳定性与合规性**：接口非官方，可能变动或受限；需多源容灾，并提示用户。
- **公开持仓数据滞后**：季度披露，自建估值有固有误差。
- **Cloudflare 限制**：Workers CPU 时长、子请求数、调用频率限制可能影响实时轮询与大区间回测。
- **localStorage 容量**：多集合 + 历史数据可能超限。

## 6. 已确认的关键决策（用户已确认 v1.0）

| # | 议题 | 最终决策 |
|---|------|---------|
| 1 | 成交净值规则 | **模拟交易遵循场外基金标准交易规则**：T 日 15:00 前申报按 T 日（最近交易日）收盘净值成交，15:00 后顺延至下一交易日；买入 T+1 确认份额，卖出 T+1 确认、资金 T+N 到账（N 可配置）；节假日/非交易日顺延。**回测**沿用历史净值逐日（按交易日）模拟。 |
| 2 | 费率 | 统一可配置费率，提供真实默认值（申购费、赎回费按持有时长分档默认、转换费），用户可改。 |
| 3 | 策略类型范围 | 预置策略模板（定投、阈值买入、止盈、止损、网格）+ 参数化配置；**不做**自由脚本/表达式。 |
| 4 | 策略冲突规则 | 同一交易日多策略触发时：**先卖后买**、同标的同方向去重/合并（设计阶段定细则）。 |
| 5 | 自建估值未覆盖仓位 | 重仓股未覆盖部分**按对应宽基指数（默认沪深300）补全**估算。 |
| 6 | 数据源选型 | 接口源：天天基金 + 蛋卷（公开接口，经 Workers 代理）；自建估算源为第三源。 |
| 7 | 导入重名冲突 | 默认**生成副本**（追加"(副本)"/序号）。 |
| 8 | 自动轮询估值 | 交易时段可选自动刷新，默认手动；自动间隔默认 60s，可配。 |
| 9 | 回测基准 | 默认"标的基金买入持有"，可选叠加沪深300 指数基准。 |

### 6.1 场外基金标准交易规则细化（决策 1 展开）
- **交易日判定**：依赖交易日历（A 股交易日历）；周末与法定节假日为非交易日。
- **申购（买入）**：
  - 申报时间 T，15:00 为分界。T 日 15:00 前 → 确认日 = T；15:00 后 → 确认日 = 下一交易日。
  - 成交净值 = 确认日收盘净值；份额 = 净申购金额 / 确认日净值（净申购金额 = 申购金额 − 申购费，采用外扣/内扣按配置，默认外扣）。
  - 份额到账（可用于卖出）= 确认日 T+1。
- **赎回（卖出）**：
  - 成交净值 = 确认日收盘净值（同上 15:00 规则）。
  - 赎回金额 = 份额 × 净值 × (1 − 赎回费率)，赎回费率按持有天数分档。
  - 资金到账 = 确认日 + N 个交易日（N 默认可配，货币基金/QDII 等差异在设计中以基金类型参数体现）。
- **转换**：A 转 B，按各自确认日净值结算，收取转换费；份额到账规则参照申购。
- **模拟时点说明**：因个人模拟工具无法获得"未来"收盘净值，实时模拟时若当日收盘净值尚未公布，则该笔交易处于"待确认"状态，待净值数据可得后回填成交结果（设计阶段定义 pending → confirmed 状态机）。
